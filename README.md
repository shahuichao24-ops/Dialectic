# Dialectic（驳真）

从第一性原理出发评估需求真伪与市场潜力的系统化方法论。

**核心流程：解构 → 建模 → 对抗性审查 → 收敛**

> "没有经过对抗性审查的结论，本质上还是猜测。"

---

## 这是什么？

Dialectic（驳真）是一个可执行的思维框架，帮助你在做产品决策前，系统性地评估一个需求/想法/方向是否值得投入。

传统的方法论只教你"如何构建"（建模型、画框架），但缺少最关键的一步——**如何摧毁你自己的结论**。Dialectic 的核心价值在于强制性的对抗性审查：假设自己错了，然后证明为什么错。

## 适用场景

- 评估一个新产品的想法是否真实
- 判断一个需求是真痛点还是伪需求
- 验证一个商业方向是否可行
- 对已有决策做逆向审查
- 分析一个产品为什么没人用

## 核心流程

```
① 解构 → ② 建模 → ③ 对抗性审查 → ④ 收敛
         ↑_______________|（循环）
```

| 阶段 | 做什么 | 输出 |
|------|--------|------|
| **① 解构** | 分解到不可再分的基本事实和隐藏假设 | 事实 vs 假设清单（含置信度标记） |
| **② 建模** | 构建可证伪的评估模型 | 带维度的评估框架（每项标注事实/假设） |
| **③ 对抗性审查** | ≥5 个独立攻击点，≥3 种证据类型，强制切换正反立场 | 漏洞清单 + Pre-Mortem |
| **④ 收敛** | 修正模型，输出带不确定性的结论 | 最终判定 + 少数派报告 |

## 使用示例

> **用户**：帮我评估一下"做一个 AI 生成周报的工具"的需求真伪
>
> **Dialectic 输出**：
> 1. 解构 → 列出事实（周报是制度性要求）和假设（用户对现状不满）
> 2. 建模 → 三维评估（痛苦×频率×替代方案不满意）
> 3. 对抗性审查 → 5 个攻击点（频率致命、付费动机薄弱、巨头已占位等）
> 4. 收敛 → 结论：伪需求。最致命问题：每周 1 次的频率无法支撑独立产品存活
